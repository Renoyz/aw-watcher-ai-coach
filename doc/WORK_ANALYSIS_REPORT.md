## 七、技术备注

### 7.1 时区问题说明

本报告第一版存在严重的时区处理错误：
- ActivityWatch 原始数据时间戳为 UTC（`+00:00`）
- 第一版分析代码直接使用 UTC 小时数进行统计
- 导致所有时间标签偏移 8 小时（如 UTC 02:00 被误标为"凌晨 2 点"，实际是北京时间上午 10 点）

**已修复**：当前版本在分析前将所有时间戳通过 `astimezone(Asia/Shanghai)` 转换为本地时间。

### 7.2 分析流程

1. 从 aw-server 拉取 window + afk 原始事件（4,086 条）
2. 时间戳 UTC → 北京时间转换
3. collector 进行 heartbeat 合并（相邻同 app <2min 拼接）
4. rule engine 基于 42 条规则分类活动类型
5. analyzer 计算专注度、深度工作、切换次数
6. 聚合生成报告

> 阶段二启用 hybrid LLM 模式后，可进一步智能识别窗口标题中的具体任务内容（如区分"写代码"和"调试"），报告精度将进一步提升。
